## 一、分类类指标：Accuracy、Precision、Recall、F1 Score

### **Accuracy**

$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$

*   TP（True Positive，真正例）：实际为正类，模型也预测为正类的样本数量。

*   TN（True Negative，真负例）：实际为负类，模型也预测为负类的样本数量。

*   FP（False Positive，假正例）：实际为负类，但模型错误地预测为正类的样本数量。

*   FN（False Negative，假负例）：实际为正类，但模型错误地预测为负类的样本数量。

这是最直接的构造方式：预测对的样本数除以总样本数。

它简单,但简单本身就是问题所在——公式里没有对"正例""负例"做任何区分,只看整体命中率。

如果数据集里 95% 的样本是负例,模型什么都不学,全部预测负例,Accuracy 照样能到 95%。这不是指标算错了,而是这个公式的设计初衷就是衡量整体正确率,没打算处理类别不均衡问题,所以类别不均衡的场景下需要换用别的指标。

### **Precision 与 Recall**

$Precision = \frac{TP}{TP + FP}, \quad Recall = \frac{TP}{TP + FN}$

这两个公式的分母不一样,是因为它们分别站在两个不同的立场上做统计。

Precision 的立场是模型自己。分母是模型预测出来的正例集合，不管有多少正例，只看模型自己挑出来的这一批里有多少是真的。所以是模型在自我检验：我预测为正的那些，准确率如何？

Recall 的立场是标准答案。分母是真实世界中所有的正例，不管模型预测了多少，只看在标准答案的正例集合里模型覆盖了多少，所以是标准答案在考核模型：所有该找出来的，你漏了多少？

这个分母上的差异导致两个指标互相拉扯：

一个模型只要把判断阈值调得极低,把绝大多数样本都判成正例,Recall 就会趋近 1,因为几乎不会漏掉真正的正例；但同时 FP 会大量增加,Precision 就会跌得很低。反过来,判断阈值调得极高,只在非常确信的时候才判正例,Precision 会很高,但会漏掉很多真正的正例,Recall 就会很低。

这两个指标要放在一起看,原因就在这里——单独看任何一个都可以通过极端策略刷到接近满分,但没有意义。

### **F1 Score**

F1 Score 是试图两边（Precision 和 Recall）不得罪、追求整体公平的裁判。

$F1 = \frac{2 \times Precision \times Recall}{Precision + Recall}$

F1 是 Precision 和 Recall 的调和平均数,而不是更直观的算术平均 $(P+R)/2$。调和平均对较小值的敏感度远高于算术平均,举个例子：

Precision=0.9,Recall=0.1,算术平均是 0.5,看起来还凑合；调和平均是

$\frac{2 \times 0.9 \times 0.1}{0.9 + 0.1} = 0.18$

只有 0.18。调和平均的数学性质决定了,只要两个数中有一个接近 0,结果就会被拉得很接近 0,不会被另一个较大的数拉高。

这正是 F1 想要的效果——逼迫模型在 Precision 和 Recall 上都不能太差,不能靠把一个刷到极端来掩盖另一个的短板。如果用算术平均,模型完全可以放弃 Recall 只优化 Precision,平均分依然能看,但调和平均不允许这种取巧。

## 二、语言模型的内在指标：困惑度

$PPL = \exp\left(-\frac{1}{N}\sum_{i=1}^{N} \log P(w_i \mid w_1, \dots, w_{i-1})\right)$

这个公式看起来复杂,但拆开看,括号里的部分

$-\frac{1}{N}\sum_{i=1}^{N} \log P(w_i \mid w_1, \dots, w_{i-1})$

其实就是训练语言模型时使用的**交叉熵损失**。它表示模型在每一步进行 next token prediction 时，对正确 token 的预测概率取负对数，再对所有位置求平均。训练过程中，这个数值几乎每天都会出现在日志里，但由于它处于对数尺度，数值变化并不直观。例如，损失从 3.2 降到 2.8，很难凭直觉判断模型到底提升了多少。

**困惑度（Perplexity）就是在交叉熵外面再做一次指数变换**： $\exp^{Cross Entropy}$,这样得到的结果可以理解为模型预测时的平均有效选择数。如果模型在每一步预测时,等价于在 $k$ 个同样可能的候选词里随机选一个,那么困惑度就等于 $k$。

PPL=1 意味着模型每一步都以概率 1 精确预测出正确的词；

PPL 等于词表大小,意味着模型完全没学到规律,退化成均匀随机猜测。

对于介于两者之间的情况，例如 PPL=3.7，可以理解为模型平均每一步的不确定性，相当于在 3.7 个等概率候选词之间做选择。这里的 3.7 并不是实际存在 3.7 个候选词，而是一个平均意义上的等效数量。

从这个角度看，困惑度并没有引入新的信息，它只是把交叉熵从对数尺度转换成了更容易理解的形式。二者描述的其实是同一件事：交叉熵更适合作为模型训练时的优化目标，而困惑度更适合作为衡量模型预测能力的评测指标。

## 三、文本相似度指标：BLEU、ROUGE

### **BLEU**

$BLEU = BP \times \exp\left(\sum_{n=1}^{N} w_n \log p_n\right)$

BLEU 是评价机器翻译好坏的指标。在了解 BLEU 之前，我们先学习一下 n-gram。n-gram 就是"连续 n 个词的切片"。n 越大，看的范围越长。例如句子："我喜欢吃苹果"：

| n | 名称                   | 切法          | 结果                | 它在检查什么？                         |
| - | -------------------- | ----------- | ----------------- | ------------------------------- |
| 1 | **1-gram / unigram** | 一个一个词切      | `我` `喜欢` `吃` `苹果` | **用词对不对**——"苹果"这个词用了吗？          |
| 2 | **2-gram / bigram**  | 两个两个词切，滑动窗口 | `我喜欢` `喜欢吃` `吃苹果` | **局部搭配对不对**——"喜欢吃"这个搭配合理吗？      |
| 3 | **3-gram / trigram** | 三个三个词切      | `我喜欢吃` `喜欢吃苹果`    | **语序和流畅度**——"我喜欢吃苹果"这个短语整体像不像人话 |
| 4 | **4-gram**           | 四个词切        | `我喜欢吃苹果`          |                                 |

BLEU 的核心部分是对 1-gram 到 4-gram(通常取 N=4)的精确率 $p_n$ 做加权几何平均,而不是算术平均。用几何平均的原因和 F1 用调和平均类似——只要某一档 n-gram 的精确率是 0(比如模型生成的四元组完全没有一个匹配参考译文),整个几何平均就会归零,因为 $\log 0 \to -\infty$。

这个设计强迫模型必须同时在词汇选择(1-gram,对应用词是否准确)和局部语序(更高阶的 n-gram,对应短语搭配是否符合参考译文的表达方式)上都有重合,不能靠单独把某一档做到完美来弥补另一档的空白。

外面乘的 BP(Brevity Penalty)是后补的修正项:

$BP = \begin{cases} 
1 & \text{if } c > r \\[6pt]
\exp(1 - r/c) & \text{if } c \leq r
\end{cases}$

其中 $c$ 是生成文本长度, $r$ 是参考文本长度。

这一项存在的原因是 n-gram 精确率偏袒短句子。精确率的公式是：

$p_n = \frac{\text{生成文本中匹配上的 n-gram 数}}{\text{生成文本中 n-gram 的总数}}$

分母是"生成文本里 n-gram 的总数"——生成得越短，分母就越小，越容易凑出高精确率。如果模型只生成一个词,而这个词恰好出现在参考译文里,精确率直接是 100%。BP 就是为了堵住这个漏洞,人为规定生成文本比参考文本短的时候要打折扣,长度越不足折扣越狠。

### **ROUGE**

BLEU 侧重于"你生成的这些内容里，有多少是对的？"（**精确率，站生成文本**）

ROUGE 侧重于"参考答案里的内容，覆盖了多少？"（**召回率，站参考文本**）

$\text{ROUGE-}N = \frac{\text{参考文本中匹配上的 n-gram 数}}{\text{参考文本中 n-gram 的总数}}$

对比 BLEU 的公式会发现,ROUGE 的分母是参考文本里 n-gram 的总数,而不是生成文本里的。

BLEU 面向翻译任务,关心的是"模型生成的内容有多少是准确的",**分母站在生成文本这边,本质是精确率。翻译的底线是"准确"。模型如果多译、错译、胡编，比漏译更致命。**

ROUGE 面向摘要任务,关心的是"参考摘要里的关键信息,模型的生成结果覆盖了多少",**分母站在参考文本这边,本质是召回率**。摘要的底线是"完整"。模型如果漏掉了原文的关键信息，比多写几句更致命。

例如：

参考文本：`我 喜欢 吃 苹果`（4 个 1-gram）

场景 A：机器生成多了（翻译中常见的"多译"）

> 生成文本：`我 喜欢 吃 苹果 和 梨 以及 香蕉`

*   BLEU-1：匹配上的 4 个 / 生成文本总共 7 个 = 4/7 ≈ 57%（低，因为多写了废话）

*   ROUGE-1：匹配上的 4 个 / 参考文本总共 4 个 = 4/4 = 100%

场景 B：机器生成少了（摘要中常见的"漏关键信息"）

> 生成文本：`我 喜欢 苹果`（漏了"吃"）

*   BLEU-1：匹配上的 3 个 / 生成文本总共 3 个 = 3/3 = 100%（如果只看这个，好像还行？）

*   ROUGE-1：匹配上的 3 个 / 参考文本总共 4 个 = 3/4 = 75%（低，因为漏了"吃"）

## 四、代码生成评测：Pass\@k

Pass\@k = 给模型 k 次机会写代码，至少有一次能通过测试的概率。

$Pass@k = \mathbb{E}_{Problems}\left[1 - \frac{\binom{n-c}{k}}{\binom{n}{k}}\right]$

**为什么不直接采样，用"生成 k 个，看有没有一个对"？**

因为抽样的随机性会完全淹没模型的真实能力，只是观测到了一个随机结果。假设模型对某道题的真实能力如下：

| 指标      | 真实概率               |
| ------- | ------------------ |
| Pass\@1 | 30%                |
| Pass\@5 | 83%（生成5个里至少1个对的概率） |

现在你用直接法测 Pass\@5：对这道题生成 5 个代码，看有没有 1 个通过。

你会得到什么结果？

*   运气好时：5 个里有 1 个对的 → 这道题得分 1

*   运气差时：5 个全错 → 这道题得分 0

看到了吗？真实能力是 83%，但你的观测结果只能是 0 或 1，完全没有中间地带。你的观测值和真实值之间差距巨大，而且纯靠运气。

**真正的解法：过采样 + 组合数学。不要只抽 k 个去赌运气，而是先大量采样，再用数学算概率。**

**步骤拆解**

**第一步：过采样**

对每道题，让模型生成 n 个候选代码（比如 n=200），然后全部跑测试，数出其中能通过的有 c 个。

比如 n=200，其中 c=60 个能通过。

**第二步：用组合数学算概率**

现在问题是：如果我从这 200 个里随机不放回地抽 k 个，至少抽到一个正确答案的概率是多少？我们再来看一下公式：

$Pass@k = \mathbb{E}_{Problems}\left[1 - \frac{\binom{n-c}{k}}{\binom{n}{k}}\right]$

| 符号               | 含义                          |
| ---------------- | --------------------------- |
| $\binom{n}{k}$   | 从 n 个样本里选 k 个的总组合数          |
| $\binom{n-c}{k}$ | 从 **错误样本**（n-c 个）里选 k 个的组合数 |
| 分式               | **抽到的 k 个全部来自错误样本**的概率      |

也就是：你运气差到爆，k 次机会全抽到了错的。

**第三步：用 1 减去"全错"的概率，就是至少有一个对的概率。**

最后对很多道题算一遍，然后取平均。

这种做法的好处是,同一批 $n$ 个采样结果可以复用来估计任意 $k$ 值的 Pass\@k,而且是无偏估计,不需要针对每个 $k$ 重新采样,方差也远小于直接采样 $k$ 个去做判断。这是 Codex 论文里提出的具体解法,核心动机就是用可复用的过采样加上解析计算,换取更稳定、更省算力的估计。

## 五、人类偏好对齐评测：Win Rate 与 Elo

### **Win Rate**

这个公式算的是胜率，即模型 A 和模型 B 打了很多场，A 赢了多少比例。

$WinRate = \frac{Wins + 0.5 \times Ties}{Total\ Comparisons}$

*   Wins：模型 A 赢的场次

*   Ties：模型 A 平局的场次

*   0.5：平局的折算系数——把一场平局拆成"半胜半负"

*   Total Comparisons：总对战次数（Wins + Losses + Ties）

*   WinRate：最终算出的胜率

这个公式本身没有太多机关,平局按半胜处理——如果把平局看作双方各有一半的胜率,这样处理能让胜率的统计期望保持无偏。它的局限在于只能反映两个模型之间的相对强弱,没法直接搬到多个模型的排行榜上,因为 A 打赢 B 60%,B 打赢 C 60%,不代表 A 打赢 C 的概率是确定的数值,胜率不具备传递性。这也是需要 Elo 的原因。

### **Elo Rating**

Elo是指，给每个模型一个"实力分数"，用分数差预测胜负概率，再根据实际结果和预测的差异来调整分数。

$E_A = \frac{1}{1 + 10^{(R_B - R_A)/400}}$

$R_A' = R_A + K \times (S_A - E_A)$

*   $R_A$：模型 A 当前评分

*   $R_B$：模型 B 当前评分

*   $E_A$：模型 A 的预期胜率

*   $S_A$：模型 A 的实际结果

*   $R_A'$：模型 A 更新后的评分

*   $K$：调整系数

*   $400$：尺度常数，是国际象棋 Elo 系统沿用下来的经验尺度,决定了评分差和胜率之间的具体换算比例。

**Elo 系统假设每个模型都有一个隐藏的"真实实力值",实际对战结果是这个实力值加上随机波动后的体现。**

$E_A$ 这个logistic函数的作用是把两个模型的评分差 $R_B - R_A$ 映射成一个 0 到 1 之间的期望胜率,分差越大,期望胜率越接近 0 或 1；分差为 0 时期望胜率正好是 0.5。

用logistic函数而不是线性映射,是因为它满足"胜率不能超过 1 也不能低于 0"这个约束,而且 $E_A + E_B = 1$ 恒成立。

更新公式 $R_A' = R_A + K(S_A - E_A)$ 的核心逻辑是：结果越出乎意料，分数调整越大。

$S_A$ 是实际结果(赢 1、输 0、平 0.5),

$E_A$ 是赛前预期,

$S_A - E_A$ 这个差值代表这场比赛的结果有多出乎意料。

一个排名靠后的模型爆冷打赢排名靠前的模型,$S_A - E_A$ 会很大,评分调整幅度也大；一个强模型按预期赢了一个弱模型,$S_A - E_A$ 接近 0,评分几乎不变。例如：

| 场景       | $S_A$（实际） | $E_A$（预期）          | $S_A - E_A$ | 含义   | 分数变化            |
| -------- | --------- | ------------------ | ----------- | ---- | --------------- |
| **爆冷**   | 1（A 赢了）   | 0.1（A 本该只有 10% 胜率） | **+0.9**    | 大冷门！ | A **大涨分**，B 大跌分 |
| **正常发挥** | 1（A 赢了）   | 0.9（A 本该 90% 胜率）   | **+0.1**    | 符合预期 | A **微涨分**，B 微跌分 |
| **正常失利** | 0（A 输了）   | 0.2（A 本该 20% 胜率）   | **-0.2**    | 略意外  | A **小跌分**，B 小涨分 |
| **被爆冷**  | 0（A 输了）   | 0.9（A 本该 90% 胜率）   | **-0.9**    | 大冷门！ | A **大跌分**，B 大涨分 |

$K$ 是控制调整速度的系数,决定了评分对新比赛结果的敏感程度。

这套机制不需要每两个模型之间打满对局,只要不断收集随机配对的对战结果,所有模型的评分会自然收敛到能互相比较的统一量纲上——这正是 Chatbot Arena 这类竞技场式评测采用 Elo 而不是直接用胜率排行的原因。

## 六、校准评测：ECE

ECE（Expected Calibration Error，期望校准误差），它是衡量分类模型"置信度是否可信"的核心指标。

在分类任务中，模型除了给出预测类别，还会输出一个置信度。一个"校准良好"的模型，其置信度应该约等于实际准确率——也就是说，如果模型对 100 个样本都输出 0.83 的置信度，那么其中大约应该有 83 个预测正确。

**ECE 就是量化"置信度"与"实际准确率"之间差距的指标。ECE 越小，模型越校准。**

$ECE = \sum_{m=1}^{M} \frac{|B_m|}{n} \left| acc(B_m) - conf(B_m) \right|$

公式的直观含义：**把样本按置信度分箱 → 每个箱子里算"实际准确率"和"平均置信度"的差距 → 按箱子大小加权求和。**

![image.png](https://p0-xtjj-private.juejin.cn/tos-cn-i-73owjymdk6/5203a854898342988409f2a052353e5f~tplv-73owjymdk6-jj-mark-v1:0:0:0:0:5o6Y6YeR5oqA5pyv56S-5Yy6IEAg54yr5ZKq6ICB5biI:q75.awebp?policy=eyJ2bSI6MywidWlkIjoiMzQyNjUzMjU4MjIzOTk5OCJ9&rk3s=f64ab15b&x-orig-authkey=f32326d3454f2ac7e96d3d06cdbb035152127018&x-orig-expires=1784991873&x-orig-sign=%2FspWES1idEFqRztTv851Sx8K%2BA0%3D)

这个公式要处理一个统计上的具体困难:模型对单个样本给出的置信度是一个连续值(比如 0.83),而这个样本的对错是一个二值结果(1 或 0)。单独拿一个样本的置信度和它的对错做比较是没有意义的——不能说"模型说 0.83 的把握,但这题错了,所以不校准",因为置信度 0.83 本来就允许有 17% 的错误率。

要验证置信度是否可信,只能把置信度接近的样本归到一组(分箱,bin),在同一组内统计实际的准确率 $acc(B_m)$,和这组样本的平均置信度 $conf(B_m)$ 做比较。

外层按 $|B_m|/n$ 加权求和,是为了让样本多的区间在总误差里占更大权重,避免样本很少的区间(统计噪声大)对最终结果影响过大,这样算出来的 ECE 才是对整个数据集上校准误差的无偏估计。

## 七、与人类评测的相关性：Pearson 与 Spearman

**Pearson 相关系数**

$r = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum (x_i - \bar{x})^2} \sqrt{\sum (y_i - \bar{y})^2}}$

分子是协方差的核心部分,衡量两组变量偏离各自均值的方向是否一致——如果 $x$ 偏大的时候 $y$ 也偏大,乘积为正,反之为负,求和之后能看出整体的同向或反向趋势。

分母是两组变量各自标准差的乘积,作用是归一化,把结果限制在 $[-1, 1]$ 之间,消除量纲和数值尺度的影响。

这个结构决定了 Pearson 只能捕捉**线性**相关——如果两组数据是严格的非线性单调关系(比如 $y = x^3$),Pearson 算出来的相关系数可能明显小于 1,尽管两者的排序关系是完全一致的。

**Spearman 相关系数**

$\rho = 1 - \frac{6 \sum d_i^2}{n(n^2 - 1)}$

Spearman 的解决办法是先把原始数值转换成排名,再对排名计算相关性,这个公式是在排名没有并列的前提下,对排名做 Pearson 相关计算后化简得到的等价形式,$d_i$ 是每个样本在两组排名中的名次差。因为排名之间的关系只反映单调性,不要求线性,所以 Spearman 对非线性但单调的关系更宽容,对异常值也更稳健——一个特别离谱的极端值在排名里只是"排第一或最后",不会像在 Pearson 里那样因为数值本身过大而拉偏整个相关系数。

在评测方法论的研究里,验证一个自动化指标是否可信,通常是拿它给一批样本打的分,和人工打分做相关性分析。这时候更常用 Spearman,原因是人工打分本身经常是有序而非严格线性的量表(比如 1 到 5 分的李克特量表),排名一致比数值上的线性对应更贴近"这个自动指标是否能反映人类的相对偏好排序"这个实际需求。
